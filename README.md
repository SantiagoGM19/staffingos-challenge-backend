# staffingos-challenge-backend

A RESTful backend API built with **AdonisJS v6** that acts as a proxy to the [JSONPlaceholder](https://jsonplaceholder.typicode.com) public API, adding custom JWT-based authentication and session management backed by a MySQL database.

---

## Tech Stack

- **Framework**: AdonisJS v6
- **Language**: TypeScript
- **Database**: MySQL (via Lucid ORM)
- **Auth**: Custom JWT (jsonwebtoken) + session persistence in DB
- **Validation**: VineJS
- **Testing**: Japa (unit + functional/integration)
- **External API**: [JSONPlaceholder](https://jsonplaceholder.typicode.com)

---

## Prerequisites

- Node.js >= 20
- npm >= 10
- MySQL server running locally (or remotely)

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/SantiagoGM19/staffingos-challenge-backend.git
cd staffingos-challenge-backend
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Copy the example env file and fill in the required values:

```bash
cp .env.example .env
```

Then edit `.env`:

```env
# Node
TZ=UTC
PORT=3333
HOST=localhost
NODE_ENV=development

# App
LOG_LEVEL=info
APP_KEY=<generate_with_node_ace_generate_key>
APP_URL=http://localhost:3333

# Database
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_DATABASE=your_db_name

# Custom
PASSWORD=your_shared_password        # The single password used for all users
JSONPLACEHOLDER_URL=https://jsonplaceholder.typicode.com
JWT_SECRET=your_jwt_secret           # Any long random string
```

> **Note:** `APP_KEY` can be generated by running: `node ace generate:key`

> **Note:** `PASSWORD` is a single shared password configured server-side. Since users are fetched from JSONPlaceholder (which has no real authentication), the API validates the user's email against JSONPlaceholder and checks this shared password.

### 4. Run the database migration

```bash
node ace migration:run
```

This creates the `user_sessions` table in your MySQL database.

### 5. Start the development server

```bash
npm run dev
```

The server will start at `http://localhost:3333`.

Tests use an **SQLite in-memory database** (configured in `.env.test`) — no MySQL setup is needed to run them.

To run only unit tests or functional tests:

## API Routes

All responses follow a consistent envelope format:

```json
{
  "status": "success" | "error",
  "message": "...",
  "data": { ... } | [ ... ] | null
}
```

---

### Authentication

#### `POST /auth/login`

Authenticates a user. Looks up the email against JSONPlaceholder's `/users` endpoint and validates the shared password. On success, creates a session and returns a JWT token.

**Request Body:**
```json
{
  "email": "Sincere@april.biz",
  "password": "yourpassword"
}
```

**Success Response** `200 OK`:
```json
{
  "status": "success",
  "message": "Login successful",
  "data": {
    "user": {
      "id": 1,
      "name": "Leanne Graham",
      "username": "Bret",
      "email": "Sincere@april.biz"
    },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

**Error Responses:**
- `401 Unauthorized` — Invalid email or wrong password
- `422 Unprocessable Entity` — Validation failed (invalid email format, password too short)

---

#### `POST /auth/logout`

Invalidates the current session. The JWT token is marked as inactive in the database.

**Headers:**
```
Authorization: Bearer <token>
```

**Success Response** `204 No Content` — empty body.

**Error Responses:**
- `401 Unauthorized` — Missing or invalid token

---

### Posts

> ⚠️ All `/posts` routes require a valid `Authorization: Bearer <token>` header. Requests without a valid active session will be rejected with `401 Unauthorized`.

---

#### `GET /posts/:userId`

Returns all posts belonging to the given user.

**Headers:**
```
Authorization: Bearer <token>
```

**Success Response** `200 OK`:
```json
{
  "status": "success",
  "message": "Posts retrieved successfully",
  "data": [
    {
      "id": 1,
      "title": "sunt aut facere repellat provident occaecati...",
      "body": "quia et suscipit suscipit recusandae...",
      "userId": 1
    }
  ]
}
```

---

#### `POST /posts`

Creates a new post.

**Headers:**
```
Authorization: Bearer <token>
Content-Type: application/json
```

**Request Body:**
```json
{
  "title": "My new post",
  "body": "This is the content of the post",
  "userId": 1
}
```

**Validation Rules:**
- `title` — required, string
- `body` — required, string
- `userId` — required, number

**Success Response** `200 OK`:
```json
{
  "status": "success",
  "message": "Post created successfully",
  "data": {
    "id": 101,
    "title": "My new post",
    "body": "This is the content of the post",
    "userId": 1
  }
}
```

**Error Responses:**
- `422 Unprocessable Entity` — Missing or invalid fields

---

#### `PUT /posts/:id`

Updates an existing post by ID.

**Headers:**
```
Authorization: Bearer <token>
Content-Type: application/json
```

**Request Body:**
```json
{
  "title": "Updated title",
  "body": "Updated content",
  "userId": 1
}
```

**Success Response** `200 OK`:
```json
{
  "status": "success",
  "message": "Post updated successfully",
  "data": {
    "id": 1,
    "title": "Updated title",
    "body": "Updated content",
    "userId": 1
  }
}
```

---

#### `DELETE /posts/:id`

Deletes a post by ID.

**Headers:**
```
Authorization: Bearer <token>
```

**Success Response** `200 OK`:
```json
{
  "status": "success",
  "message": "Post deleted successfully",
  "data": null
}
```

## JSONPlaceholder Integration

This API does **not** store any users or posts in the local database. Instead, it proxies all CRUD operations to the public [JSONPlaceholder](https://jsonplaceholder.typicode.com) API.

The following JSONPlaceholder endpoints are called internally:

| Local Route | HTTP Method | JSONPlaceholder Endpoint | Purpose |
|---|---|---|---|
| `POST /auth/login` | `GET` | `/users?email={email}` | Fetch user by email to authenticate |
| `GET /posts/:userId` | `GET` | `/posts?userId={userId}` | Fetch all posts for a user |
| `POST /posts` | `POST` | `/posts` | Create a new post |
| `PUT /posts/:id` | `PUT` | `/posts/{id}` | Update an existing post |
| `DELETE /posts/:id` | `DELETE` | `/posts/{id}` | Delete a post |

> **Important:** JSONPlaceholder is a fake/mock API. Create, update, and delete operations will return simulated responses but no data is actually persisted on their server.
